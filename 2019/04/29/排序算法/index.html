排序算法

Posted on 2019-04-29

冒泡排序
冒泡排序，顾名思义，就像鱼从水底吐泡泡一样， 泡泡是从底逐渐浮到水面的，且升到水面的泡泡是最大的。故，冒泡排序的思想是比较两个相邻元素，选出较大的元素，放至后方，再与下一个相邻的元素进行比较，依次比较，直至比较完整个要排序的元素列，这样下来的话，元素列末尾的元素是最大的。我们将最大的元素排除在外，再针对所有的元素进行上述比较，则第二大的元素也被找出来了，以此类推，最终的元素列为递增序列。

平均时间复杂度： O(n2)

核心代码：

for(int i = numArr.length;i>=0;i–){
for(int j = 0;j <i;j++){
if(numArr[j] > numArr[j+1]){
int tem = numArr[j];
numArr[j] = numArr[j+1];
numArr[j+1] = tem;
}
}
}

由于两个相邻元素相等时，没有交换位置，故冒泡排序是一种稳定的排序算法。

插入排序
插入排序，是讲待排序的元素插入到已排好序的元素列中，其思想与冒泡排序类似，我们从元素列的第二项开始与第一项比较，排好序后，取第三项进行比较，排序，再取第四项，以此类推，把每一项与前面的有序元素列比较，比较至当前位置的前一项比待排序大，比后一项小时，进行插入操作，直至比较完整个元素列。

平均时间复杂度： O(n2)

核心代码：

for (int i = 1,l = numArr.length ; i0;j–){
if (numArr[i] < numArr[j-1]){
int tem = numArr[j-1];
numArr[j-1] = numArr[i];
numArr[i] = tem;
}
}
}

插入排序也没有相等元素的交换位置，故也是一种稳定的排序算法。

选择排序
选择排序的思想是，在每一趟从待排序的元素列中选择最小（或最大）的一个元素作为首元素，然后去除当前元素，在剩下的元素找到最小（或最大）的元素作为剩下元素列的首元素，直到所有元素排完为止。

平均时间复杂度： O(n2)

核心代码：

for (int i = 0;i<numArr.length;i++){
minIndex = i;
for (int j = i+1;j<len-1;j++){
if (numArr[minIndex]>numArr[j]){
minIndex = j;
}
}
int tem = numArr[i];
numArr[i] = numArr[minIndex];
numArr[minIndex] =tem;
}

相比于冒泡排序和插入排序，选择排序不同之处在于多了一个最小值的索引，来定位最小值所在位置，来与首元素进行交换。

正因涉及到元素的交换，移动了顺序，显然，选择排序不是一种稳定的排序算法。

快速排序
快速排序是对冒泡排序的一种改进，回想冒泡排序，是通过前后元素的不断对比来进行排序，而快速排序的方法更巧妙，通过一趟排序将要排序的数据分割成独立的两部分，其中一部分的所有数据都比另外一部分的所有数据都要小，然后再按此方法对这两部分数据分别进行快速排序，整个排序过程可以递归进行，以此达到整个数据变成有序序列。

平均时间复杂度：θ(nlogn)

核心代码：

快速排序是一种不稳定的排序算法。
